#2- Display: Inline vs. Inline-block

February 7, 2016

The Display Property

In CSS, the display property determines how an element is displayed on a webpage. Like all CSS properties, display properties are each represented by a value. For example, a display value of none hides an element from the page visually and spatially, as if it weren't there. It would be written as: 

display:none;
and would return nothing on the page. The none value is useful in combination with Javascript when you want access to an element but don't want it to take up any space on your website.

Inline and Block

The two default values of the display property are inline and block, depending on the element type. The inline and block codes deal with how much width an element takes up on the page. The inline value makes it so that each element takes up only the width of its actual size. Elements stick to and follow each other as if they are in a line. When you don't want elements sticking together, you can make the display value block, which will occupy all the available width around the element, essentially causing a line break.

Inline vs. Inline-block

There are drawbacks to both inline and block elements, however. With an inline element, you can't adjust the height or width. You also can't place block elements next to each other because the value causes a line break. In comes the inline-block value, an inline element that acts like a block element. Inline-block recognizes height and width and allows for adjacent block placement. It's a best-of-both worlds situation where you have more freedom to edit the properties of the element.
